The Plan

Static Frontend + TN12 Node. No Backend.

The BlockDAG is the server.

The Goal

Run The DAG Gate as a static frontend that connects directly to a Kaspa TN12 node via wRPC WebSocket. Game state lives on-chain as covenant-enforced UTXOs. No backend server. No database. No middleware. The browser talks to the BlockDAG.

Phase 2: On-Chain Game State

Game state moves from localStorage to covenant UTXOs. The browser builds and submits real transactions.

Browser Transaction Flow

kaspa-wasm → Rust→WASM SDK loaded in browser for tx building + signing
Keypair → Generated in browser, stored in localStorage, funded via faucet
Build Tx → Construct covenant spend with proper inputs/outputs/scripts
Submit → SubmitTransaction via same wRPC WebSocket
Read State → GetUtxosByAddresses to decode Player covenant UTXO

SilverScript Covenants

🛡

player.ss

Self-preserving UTXO. Holds character state: name, class, level, XP, HP, stats, gold, equipment. Every spend recreates the UTXO with updated state. Auth-bound to player's pubkey.

⚔

game.ss

Episodic covenant. Created when entering the forest, consumed when combat resolves. ICC with Player covenant in a single atomic transaction. Monster selection deterministic from block hash.

🏪

shop.ss

Persistent shared state. Holds inventory and prices. ICC with Player covenant: atomic tx validates gold transfer + item grant. No trusted shopkeeper.

🏟

arena.ss

PvP with staked KAS. Two Player covenant inputs + Arena covenant. Timeout via this.age > 300 (5 min per turn). Winner gets the pot.

ZK Proof Generation

Complex combat math runs off-chain in a RISC Zero guest program. Input: player stats, monster stats, random seed from block hash. Output: valid state transition (new HP, XP, gold). The proof is submitted alongside the covenant spend — the L1 verifies without re-executing. For simpler math, inline SilverScript validation avoids proof overhead entirely.

Phase 3: Multiplayer

🌍

Shared World

Player UTXOs visible to all — leaderboard by scanning covenant UTXOs. PvP challenges reference opponent's Player UTXO directly. Opponent notified via UTXO subscription.

⏰

Protocol-Native Mechanics

Daily fight reset: require(this.age > 86400) on Player UTXO — no server clock needed. Gold = locked test KAS (sompi). PvP stakes: both players lock KAS, winner takes pot.
